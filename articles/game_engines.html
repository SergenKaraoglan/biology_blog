The Game Loop

Every game engine, from the simplest 2D platformer to a colossal open-world RPG, is driven by one fundamental construct: the game loop. Each iteration of the loop processes input, updates the world state, and renders the frame. It is the heartbeat of any interactive application.

A naive loop ties physics updates to the frame rate — meaning your game runs faster on powerful hardware and slower on weak machines. The solution is to decouple the simulation from the render clock using a fixed timestep for physics and a variable timestep for rendering. This guarantees deterministic, stable physics regardless of the frame rate.

Entity-Component-System

Traditional object-oriented game architectures create deep inheritance hierarchies — a Player extends Character extends Entity. This quickly becomes rigid and fragile. The Entity-Component-System (ECS) pattern replaces inheritance with composition.

An Entity is nothing more than a unique ID. Components are pure data containers (Position, Velocity, Sprite). Systems are the logic — they query for entities that possess specific component combinations and operate on them. This data-oriented design is cache-friendly, parallelisable, and infinitely flexible.

Spatial Partitioning

Checking every object against every other object for collisions is O(n²) — disastrous at scale. Spatial partitioning structures like the Quadtree recursively subdivide 2D space into four quadrants. Each node holds a limited number of objects; when the limit is exceeded, the node splits.

Collision queries become localised: instead of testing against every object in the world, you only test against objects in the same (or neighbouring) quadrant. This transforms broad-phase collision detection from O(n²) to approximately O(n log n).

The Physics Engine

A physics engine simulates Newtonian mechanics: F = ma. Each frame, forces (gravity, friction, collisions) are accumulated, integrated via semi-implicit Euler integration to update velocities and positions, and then collision response is applied to prevent interpenetration.

When two rigid bodies collide, the engine computes a collision normal and applies an impulse — an instantaneous change in velocity — governed by the coefficient of restitution (bounciness). A restitution of 1.0 is perfectly elastic; 0.0 is perfectly inelastic.

The Scene Graph

A scene graph is a tree structure where each node represents an object in the world. Every node stores a local transform (position, rotation, scale) relative to its parent. The world transform is computed by concatenating all transforms from the root down to the node.

This hierarchical design means moving a parent automatically moves all its children. A character's arm swings relative to the shoulder; the sword swings relative to the hand. This pattern is fundamental to skeletal animation, UI layouts, and any system with nested spatial relationships.
