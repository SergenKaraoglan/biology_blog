The Heart of the Machine

The PlayStation's brain is a MIPS R3000A running at 33.8688 MHz — a 32-bit RISC processor with a classic five-stage pipeline. Every clock cycle, an instruction enters the pipeline and advances through Fetch, Decode, Execute, Memory Access, and Write-Back. When the pipeline is full, the CPU completes one instruction per cycle, achieving throughput that its raw clock speed alone could never explain.

The R3000A has 32 general-purpose registers (R0 is hardwired to zero), a 4 KB instruction cache, and a 1 KB data scratchpad. It lacks a hardware multiply/divide unit in the traditional sense — instead, multiply and divide run in a separate co-processor (CP0) and take multiple cycles, writing results to special HI and LO registers. Branch instructions carry a delay slot: the instruction immediately after a branch always executes, a quirk of the MIPS architecture that compilers and programmers must carefully manage.

The Maths Co-Processor

The PS1 could never have rendered 3D worlds with the R3000A alone. Enter the Geometry Transformation Engine (GTE), a co-processor (CP2) wired directly into the CPU that performs fixed-point matrix and vector operations at blistering speed. Where the CPU might take dozens of cycles to multiply a 3×3 matrix by a vector, the GTE completes the same operation in as few as 15 cycles.

The GTE handles the critical 3D pipeline: rotation, translation, perspective projection, depth cueing, and light-source calculation. All arithmetic is performed in fixed-point 1.15.16 or 1.3.12 format — the PS1 has no floating-point unit whatsoever. Vertices are expressed in a local coordinate space, transformed by the GTE into screen coordinates, and then handed off to the GPU for rasterization. This divide-and-conquer approach is the secret to the PS1's polygon throughput — roughly 360,000 flat-shaded polygons per second.

The Address Map

The PS1 addresses a 4 GB virtual space through the MIPS architecture's three memory segments: KUSEG (user, cached), KSEG0 (kernel, cached), and KSEG1 (kernel, uncached). In practice, most code runs from KSEG0 starting at 0x80000000, which maps to the physical 2 MB of main RAM at 0x00000000.

Beyond main RAM, the address bus routes to 1 MB of VRAM (accessible only through GPU DMA), a 1 KB scratchpad used as fast on-chip data memory, the 512 KB BIOS ROM at 0x1FC00000, and a constellation of I/O ports (GPU, SPU, CD-ROM, controllers, timers) mapped into the 0x1F800000–0x1F802000 range. Understanding this address map is the first step in emulating the PS1: every memory read or write must be decoded and routed to the correct hardware device.

Painting with Polygons

The PS1 GPU is a dedicated 2D rasterization engine — it has no concept of 3D at all. It receives commands from the CPU via a 64-byte command FIFO or through DMA, with each command specifying a primitive to draw: triangles, quads, lines, or rectangles, with optional flat shading, Gouraud (vertex) shading, texture mapping, or semi-transparency.

The GPU writes directly into 1 MB of VRAM, organized as a 1024×512 pixel framebuffer at 16 bits per pixel. Textures, CLUTs (colour lookup tables), and display buffers all share this same VRAM. Rendering order is controlled by an Ordering Table — a linked list of GPU command packets sorted by depth, which the CPU constructs each frame and then fires to the GPU via DMA in a single burst. This is how the PS1 achieves its characteristic rendering: no Z-buffer, just careful per-polygon sorting.

Digital Voices

The Sound Processing Unit is a self-contained audio DSP with 24 hardware voices, each capable of playing back ADPCM-compressed samples at variable pitch. The SPU has its own 512 KB of Sound RAM, separate from main memory, into which the CPU uploads sample data via DMA. Each voice has a programmable ADSR envelope (Attack, Decay, Sustain, Release) — the same synthesis model used in classic analogue synths.

Beyond sample playback, the SPU provides a hardware reverb unit with configurable delay, feedback, and wet/dry mix, capable of simulating environments from small rooms to cathedrals. Voices can also be configured for noise generation and frequency modulation (using one voice's output to modulate another's pitch). The final mix is output as 44.1 kHz 16-bit stereo PCM — CD-quality audio that was a major selling point of the console.
